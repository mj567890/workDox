# 设计决策记录

## 1. 文档组织方式：属性化 vs 文件夹

**决策**：属性化组织，不设用户手动创建文件夹。

**方案**：文档通过"业务事项 + 文档分类 + 标签"三维定位。左侧分类树是系统自动聚合的视图，不是物理目录。

**理由**：
- 一份文档可能同时属于"教学检查"和"2026年度材料"，文件夹方式无法解决多维度归属
- 非技术用户往往不擅长维护文件夹层级，容易越用越乱
- 属性化方式使得搜索和筛选更精确

**参考**：MacOS 的标签系统、Gmail 的标签体系

## 2. 版本管理模型：借鉴 TortoiseSVN

**决策**：非破坏式版本管理 + 明确正式版标记 + 编辑锁定

**TortoiseSVN 启发**：
| TortoiseSVN 概念 | 本系统等价物 |
|---|---|
| 图标叠加层（状态可视化） | 文档列表颜色徽标：草稿/正式版/已签批/已归档/锁定中 |
| Lock-Edit-Unlock | 编辑锁定机制：锁定 → 下载编辑 → 上传新版本 → 自动解锁 |
| 每次提交生成新版本号 | 每次上传生成 v1/v2/v3... |
| Show Log | 版本历史时间线面板 |
| Diff 对比 | 版本差异对比（后续扩展） |
| 冲突提示 | "在你编辑期间已有新版本 v4" |

**不照搬的地方**：
- 不强制 checkout→commit 流程，改为更直观的"编辑→下载→上传新版本"
- 使用"正式版"概念而非"最新版"，符合办公场景
- 版本号按文档独立编号（v1, v2...）而非全局递增

## 3. 文档编辑工作流

**决策**：MVP 阶段采用"下载-编辑-回传"模式，后续扩展到在线编辑。

### 用户操作完整流程

1. **预览确认** → 在线预览文档，确认是需要编辑的文件
2. **点击编辑** → 系统弹出选择：
   - 「锁定并下载」（推荐，防止冲突）
   - 「仅下载」（不锁定）
3. **本地编辑** → 用户在自己电脑上用 Office 软件修改
4. **回传系统** → 文档页面显示黄色提示条"你正在编辑此文档"
   - 点击「上传新版本」→ 填写变更说明 → 生成新版本号 → 自动解锁
   - 或点击「放弃编辑」→ 手动解锁
5. **通知协作人** → 事项内所有人收到"xxx更新了文档至vN"

### 锁定机制细节

- 锁定超时：24小时自动解锁
- 锁定人对所有人可见（文档列表显示🔒标识+人名）
- 非锁定人尝试编辑时提示"正在被xxx编辑，已编辑x小时"
- 支持"提醒对方尽快完成"按钮
- 支持管理员强制解锁

### 冲突处理

- 用户上传新版本时，系统检测是否有中间版本
- 如有，提示"自你下载后已有新版本vN，确认覆盖？"
- 建议用户先查看最新版本再决定

### 未来扩展

- 后续阶段集成 OnlyOffice 实现在线协同编辑
- 支持版本差异对比

## 4. 文档共享模型

**决策**：事项自动共享为主（80%），手动共享为辅（20%）。

**三种共享层级**：

| 层级 | 方式 | 适用场景 |
|---|---|---|
| 事项内自动共享 | 加入事项即自动可见所有文档 | 日常协作（默认） |
| 跨事项引用 | 手动关联到其他事项（只读） | 一个文档复用到另一个事项 |
| 指定人员共享 | 分享给非事项成员 | 跨部门共享 |

**理由**：消除"忘记发送""找不到文件""版本不一致"的问题。大部分日常协作中，用户不需要额外操作，文档自然在事项内流转。

## 5. 技术栈选型

| 层次 | 选择 | 替代方案 | 选型理由 |
|---|---|---|---|
| 后端 | Python FastAPI | Java Spring Boot | 开发效率高，Python 文档处理生态（python-docx/openpyxl/PyPDF2）强 |
| 前端 | Vue 3 + Element Plus | React + Ant Design | 国内生态成熟，Element Plus 对管理后台支持好 |
| 数据库 | PostgreSQL | MySQL | 自带全文检索，JSON 支持好 |
| 文件存储 | MinIO | 本地存储/云OSS | S3兼容，开源，后期可无缝迁移到云 |
| 文档转换 | LibreOffice + PDF.js | OnlyOffice | 开源免费，无商业许可限制 |
| 异步任务 | Celery + Redis | RabbitMQ/Kafka | 轻量，适合部门级系统 |

## 6. 分阶段实施策略

| 阶段 | 内容 | 优先级 |
|---|---|---|
| Phase 1 | 文档集中管理 + 在线预览 | 解决核心痛点"文档找不到" |
| Phase 2 | 业务事项 + 协同共享 | 解决"重复劳动、进度不透明" |
| Phase 3 | 流程管理 + 防漏提醒 | 解决"长线工作漏环节" |
| Phase 4 | 领导驾驶舱 | 解决"领导无法掌握进度" |
